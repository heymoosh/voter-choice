## Technical Context

### Tech Stack

- **Framework**: Next.js 15 (App Router) with TypeScript
- **Styling**: Tailwind CSS 4
- **Testing**: Vitest (unit), Playwright (e2e)
- **i18n approach**: Custom React context + typed dictionaries (no external i18n library needed)
- **Storage**: localStorage for language preference

### Architecture Decision

**Custom i18n context with typed translation dictionaries.**

- No external library (next-intl, react-i18next) needed — the translation surface is fixed and well-defined
- Dictionaries are TypeScript objects with a shared type, enabling compile-time coverage checking
- Provider wraps the client component tree; hook `useTranslation()` used throughout components

**Research: i18n library vs custom**
- Decision: Custom typed dictionaries
- Rationale: Fixed two-language scope, no routing needed, full type safety, zero bundle overhead from library
- Alternatives considered: next-intl (requires URL routing patterns), react-i18next (adds 60KB+, overkill for static strings)

**Research: localStorage vs URL vs server state**
- Decision: localStorage
- Rationale: PHASE2_SPEC explicitly requires localStorage; URL params and server state are out of scope
- Storage key: `voter-choice-language` (values: `"en"` | `"es"`)

**Research: lang attribute update**
- Decision: Small client component `HtmlLangUpdater` inside layout that calls `document.documentElement.lang = locale` via useEffect
- Rationale: layout.tsx is a Server Component; minimal client footprint avoids converting it entirely

**Research: Spanish prompt storage**
- Decision: Complete standalone Spanish translation as a TypeScript constant in `src/lib/ballotPrompt.es.ts`
- Rationale: PHASE2_SPEC §Part 1 explicitly requires "stored as a complete translation, not generated by string interpolation on fragments"

## Key Design Decisions

### Translation Dictionary Shape

```typescript
// src/lib/i18n/types.ts
export interface Translations {
  hero: { headline: string; subtitle: string; }
  chatbots: { linkLabel: string; }
  zipForm: { label: string; placeholder: string; submitButton: string; }
  errors: {
    emptyZip: string;
    invalidZip: string;
    zipNotFound: string;
    multiState: string;
    deadlinesPassed: string;
    noElections: (state: string) => string;
  }
  stateInfo: {
    title: string; election: string; date: string;
    registrationDeadlines: string; online: string; byMail: string; inPerson: string;
    postmark: string; received: string;
    earlyVoting: string; from: string; through: string; notAvailable: string;
    voterId: string; required: string; notRequired: string; acceptedIds: string;
    phonesAtPolls: string; sampleBallot: string; countyOffice: string;
  }
  deadline: {
    passed: string;
    daysLeft: (n: number) => string;
  }
  prompt: {
    instructions: string;
    copyButton: string;
    copiedButton: string;
    fallbackInstructions: string;
  }
  tips: {
    heading: string;
    items: string[];
  }
  footer: {
    shareHeading: string;
    shareText: string;
    attribution: string;
  }
  stateSelector: { prompt: string; }
  loading: string;
  notFound: { heading: string; message: string; link: string; }
  accessibility: { skipToContent: string; languageChanged: string; }
}
```

### Context Block in Spanish

Labels translated; injected data values (state name, dates from JSON, URLs) unchanged. Date values formatted using `formatDate(date, 'es')` for the Spanish format (e.g., "3 de marzo de 2026").

### Language Toggle Placement

Fixed-position element in the top-right corner of the viewport (CSS: `fixed top-4 right-4 z-50`), ensuring visibility at all scroll positions and on mobile without requiring a menu.
