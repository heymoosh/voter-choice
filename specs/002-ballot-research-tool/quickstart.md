## Key development notes

- **TDD required**: Write failing test в†’ run `npm test` to confirm red в†’ implement в†’
  confirm green. Never write implementation before test.
- **Test files**: Co-located in `src/` as `*.test.ts` / `*.test.tsx`
- **Data-testid attributes**: All 13 required IDs from `docs/PROJECT_SPEC.md` must be
  present. The shared Playwright suite in `e2e/ballot-tool.spec.ts` tests them.
- **Clipboard in tests**: Mock `navigator.clipboard.writeText` in Vitest unit tests.
  Playwright e2e tests use `page.evaluate` to check clipboard where needed.
- **Date determinism**: `getDeadlineStatus` accepts `todayISO` as a parameter (default:
  `new Date().toISOString().split('T')[0]`). Tests inject a fixed date.
- **BALLOT_PROMPT.md**: Import as a string at build time using Next.js's `fs` in a
  Server Component or embed as a TypeScript constant (preferred for simplicity).
